# 🐱 猫咪归位 (Cat Finder) — 微信小游戏

一款基于 **Canvas 2D** 的逻辑解谜微信小游戏,使用纯 JavaScript 实现,无第三方依赖。玩家需要在不同尺寸的彩色棋盘上,根据规则找到每一只猫咪的"家"。

> **平台**:微信小游戏 / 微信开发者工具
> **引擎**:原生 Canvas 2D + 微信小游戏 API
> **AppID**:`wxc405b40e5037914a`

---

## ✨ 特色亮点

- 🎨 **商业化 UI 设计**:渐变背景、玻璃拟态卡片、弹性缩放弹窗、动态粒子背景
- 🧠 **自动关卡生成**:基于回溯求解器动态生成可解的关卡,关卡布局每次刷新都不同
- ♿ **无障碍支持**:**色盲模式**为每种颜色配置独立形状标识

- 📱 **响应式适配**:750×1334 设计稿,自动按设备分辨率等比缩放

## 🎯 游戏规则

在 N×N 的彩色棋盘上(每种颜色恰好 N 格),为每一只猫咪找到它的"家",使其满足以下**全部规则**:

| 规则 | 说明 |
|------|------|
| ✅ 规则 1 | 每**行**只能有 1 只猫 |
| ✅ 规则 2 | 每**列**只能有 1 只猫 |
| ✅ 规则 3 | 每种**颜色**只能有 1 只猫 |
| 🚫 规则 4 | 猫的**上下左右**相邻格不能有其他猫 |

找出全部 N 只猫即可通关。

## 🧩 核心模块说明

### 1. 关卡生成(`LevelGenerator` + `CatSolver`)
- **算法流程**:
  1. 随机选 N 个种子点 → 波浪式扩散生成 N 个连通区域
  2. 验证每种颜色恰好 N 格 + BFS 连通性 + 避免整行/整列同色
  3. 调用 `CatSolver` 求解,验证至少存在一组合法解
  4. 失败时使用 Latin Square 兜底布局
- **求解策略**:回溯搜索(带 100ms 超时)→ 失败转贪心 → 极端情况线性扫描

### 2. 触摸路由(`GameManager._handleTouch`)
- 通过 `_lastTouchTime` / `_touchStartCell` / `_lastTapCell` 三个状态机区分
  - **点击**(< 300ms 且非双击):切换标记
  - **长按**(> 300ms):切换标记
  - **双击**(< 300ms 内同一格子再次点击):放置/移除猫

### 3. 存档系统(`SaveSystem`)
- 键名:`cat_finder_save`,通过 `wx.getStorageSync` / `wx.setStorageSync` 持久化
- 字段:
  - `currentLevel` / `unlockedLevel`:当前 / 已解锁关卡
  - `streak` / `bestStreak`:当前 / 历史最佳连胜
  - `bestTimes`:各尺寸(4~8)最佳通关时间(分钟)
  - `firstClearTime`:毕业证书(首次通关时间,仅记录首次)
  - `totalPlays`:累计通关次数
  - `settings`:音乐/音效/震动开关

### 4. 跨环境全局注册(`game.js` 中的 `expose`)
- 微信小游戏每个 js 文件是独立作用域,子模块直接引用 `SaveSystem` / `UI` 等未限定名
- 主入口把所有共享标识符挂到 `GameGlobal` / `global` / `window`,保证子模块函数体执行时能解析

## 🛠️ 技术栈

- **JavaScript**(ES5 兼容语法,可在小游戏 jscore 中运行)
- **Canvas 2D**(所有渲染)
- **微信小游戏 API**:`wx.createCanvas` / `wx.onTouchStart` / `wx.getStorageSync` / `wx.createInnerAudioContext` / `wx.vibrateShort` / `wx.showToast` 等
- **算法**:回溯搜索 + 贪心 + Latin Square
- **存储**:本地 `wx.setStorageSync`(单 key JSON)
